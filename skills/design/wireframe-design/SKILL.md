# wireframe-design — ブラウザで触れる低忠実度ワイヤー

## 概要

画面設計を「**ブラウザで開けてクリック遷移できる低忠実度の HTML ワイヤーフレーム**」＋「散文の `screens.md`」として残す手法。
配色やタイポは後回しにし、**構造・要素の有無・操作フロー**だけを早く固める。図ツールも画像も使わず、単一HTML（Tailwind CDN）なので作成も共有も軽い。

**中核原則**: ワイヤーは「触って流れを検証する道具」。静的な絵やプロセ説明ではなく、実際にクリックして画面遷移を試せる状態にする。

## いつ使うか

- 実装前に画面構成・画面遷移・状態バリエーションを固めたいとき
- 「画面設計」「ワイヤー」「モック」「UIのたたき台」を求められたとき
- 既存UIの改修案を、触れる形でレビューに出したいとき

**使わない**: 振る舞い・ロジックの仕様そのもの（それは spec 側に書く。ここは「見た目・操作」に絞る）。

## 成果物（`docs/3_design/` 等に置く）

| ファイル | 役割 |
| --- | --- |
| `screens.md` | 画面一覧（役割＋仕様リンクの表）／遷移図（mermaid）／状態バリエーション／**未確定**セクション |
| `wireframes/index.html` | 全ワイヤーへのリンク一覧（入口） |
| `wireframes/NN-画面-状態.html` | 1画面1状態のワイヤー。相互リンクでクリック遷移 |

テンプレは [templates/](templates/) にある。複製して埋める:
- [templates/screens.md](templates/screens.md)
- [templates/index.html](templates/index.html)
- [templates/screen.html](templates/screen.html) — **デスクトップ/Web 向け**雛形（サイドナビ＋本体の2ペイン、1280px 枠）
- [templates/screen-phone.html](templates/screen-phone.html) — **モバイル向け**雛形（下部タブ＋縦1カラム、375px 枠）

案件の対象ビューポートに合う方を複製する。枠クラスを替えるだけでなくレイアウトごと変わるため、phone⇄desktop は別雛形を使い分ける。

## 手順

1. **画面と遷移を洗い出す** → `screens.md` に画面一覧表（役割＋仕様リンク）と mermaid の遷移図を書く。
2. **各画面の状態を列挙** → 空 / 読込中 / 失敗 / フォールバック / 権限なし 等。`screens.md` の「状態バリエーション」に明記。状態は粒度で扱いを変える（下表）。
3. **ワイヤーを作る** → `templates/screen.html` を複製し `wireframes/NN-画面-状態.html` に。デバイス枠を選び、レイアウトと注釈幅も枠に合わせる。同一画面で複数の画面レベル状態を見比べるときは `<section>` を複製して横並びにする。
4. **クリック遷移を張る** → `index.html` から各画面へ、画面間も `href`/`onclick` で実際に遷移できるようにする。
5. **低忠実度を保つ** → 配色・タイポは「未確定」のまま構造だけ。決めたら `screens.md` と各HTMLの説明文に反映する。
6. **spec と相互リンク** → 各ワイヤーから対応する仕様ファイルへリンク。振る舞いの正は spec、ワイヤーは見た目・操作。

## アイコン

素の Tailwind にアイコンは無い。OS 依存グリフ（SF Symbols 等）は他環境で描画されないので使わない。テンプレは **Iconify** を CDN 1本で読み込み済み:

```html
<script src="https://cdn.jsdelivr.net/npm/iconify-icon@2/dist/iconify-icon.min.js"></script>
<!-- 色/サイズは Tailwind の text-* が効く（既定 1em・currentColor）。初期化呼び出し不要。 -->
<iconify-icon icon="lucide:settings" class="text-lg text-neutral-500"></iconify-icon>
```

`lucide:`（線アイコン、低忠実度向き）を既定に、必要なら `mdi:` `material-symbols:` 等 [icon-sets.iconify.design](https://icon-sets.iconify.design) から選ぶ。オフライン重視なら Lucide 単体、塗り/ブランド系は Font Awesome でも可。

## 守ること（rationale つき）

- **低忠実度の規律**: 早い段階で色・フォントを決めると、構造の議論が止まり手戻りが増える。「未確定」と書いて構造に集中する。決めた項目だけ明示的に「確定」と書く。
- **状態を網羅**: 正常系だけのワイヤーは後で空・失敗で破綻する。各画面の異常系を最初に列挙して別フレーム or 注釈で示す。
- **クリックできること**: 流れの良し悪しは触らないと分からない。`onclick`/`href` で必ず遷移可能にする（パラメータは `URLSearchParams` で簡易に渡す）。
- **spec と分離**: ワイヤーに振る舞いの詳細を書き込むと、どちらが正か分からなくなる。ワイヤー＝見た目・操作、spec＝振る舞い、で相互リンクする。
- **デバイス枠の明示**: ビューポート（phone 375 / tablet 768 / desktop 1280 等）を枠で固定して、対象サイズ前提でレイアウトを見る。
- **注釈を添える**: 各フレーム横に monospace の note で「この要素の出し分けルール」を1〜3行書く。レビューで意図が伝わる。
- **状態の粒度で扱いを分ける**: **画面レベル状態**（空 / 読込中 / 失敗 / 権限なし）は別フレーム（別HTML）に分ける。**要素レベル状態**（入力エラー / モーダル / トースト / インライン展開）は遷移を伴わないので、同一フレーム内で併記するか JS で開閉して見せる。「異常系＝必ず別フレーム」と機械的に分けない。
- **共通ナビ/ヘッダ**: 全画面共通のグローバルナビは、低忠実度では DRY より閲覧性を優先して各フレームにコピーしてよい（後で実装側で共通化する前提）。

## よくある失敗

| 失敗 | 直し方 |
| --- | --- |
| いきなり高忠実度（色・影・フォント作り込み） | 構造と要素の有無だけ。色は neutral 系のまま「未確定」と明記 |
| 正常系1枚だけ | 状態を列挙し、空/読込中/失敗を別フレームに |
| 静的な絵・箇条書き説明だけ | 単一HTMLにして相互リンクでクリック遷移可能に |
| 仕様の文章をワイヤーに書き込む | 振る舞いは spec に。ワイヤーからはリンクするだけ |
| ファイル名がバラバラで一覧できない | `NN-画面-状態.html` 連番＋`index.html` で入口を1つに |
| デバイス枠クラスだけ替えてレイアウトは縦積みのまま | 枠に合わせてレイアウト（phone=縦1カラム / desktop=サイドナビ＋本体・テーブル）と注釈幅も変える |
| 入力エラーやモーダルを別フレームに切り出して画面が増殖 | 要素レベル状態は同一フレーム内で（前述の粒度ルール） |
